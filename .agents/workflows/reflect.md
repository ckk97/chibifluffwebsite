#  Workflow: The /reflect Sync

## 1. Purpose
To automate the transition of ephemeral session data into permanent project intelligence. This workflow manages the **Extract** and **Retrieve** phases, ensuring `MEMORY.md`, `CONTEXT.md`, and `JOURNAL.md` are always synchronized with the latest frontend developments.

## 2. The Reflection Protocol (The 5-Step Loop)

### Phase 1: Extract (The Harvest)
*   Scan the conversation for finalized **React Native** components and **Anime.js** logic.
*   Identify new "Design Tokens" (e.g., custom easing, specific hand-drawn border ranges).
*   Isolate "Learned Lessons" regarding performance or IDE-specific behavior.

### Phase 2: Update MEMORY.md (Long-Term)
*   **Golden Code:** Append perfected snippets to the code repository.
*   **Brand Soul:** Refine existing design tokens if the aesthetic has evolved.
*   **Performance:** Log any 60fps benchmarks or frame-drop triggers discovered.

### Phase 3: Update CONTEXT.md (Current State)
*   **Roadmap:** Move completed tasks to the "Done" section.
*   **Status:** Update the active development goal and system health (🟢/🟡/🔴).
*   **Dependencies:** List any new frontend libraries added.

### Phase 4: Update JOURNAL.md (The Log)
*   Generate a concise daily entry.
*   Document the **Execute, Evaluate, Extract, Refine, Retrieve** notes for the session.

### Phase 5: Retrieve (The Handover)
*   Generate three "Next Steps" for the following session.
*   Define the immediate priority to ensure a seamless "Retrieval" of momentum.

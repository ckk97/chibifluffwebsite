# Chibi Fluff: Frontend Agentic Constitution

## 1. Project Mission
To build a high-fidelity, high-performance frontend ecosystem for the **Chibi Fluff** brand using **React Native (Expo)** and **Anime.js**. The goal is to translate a hand-drawn, pet-centric aesthetic into a fluid, 60fps digital experience optimized for Apple Silicon (M5).

## 2. The Core Operational Loop
The Agent must apply these five rules to every interaction without exception:

*   **Execute:** Perform development tasks (UI components, animation timelines, styling) using the established tech stack. Every code block must be functional and modular.
*   **Evaluate:** Audit all output for performance bottlenecks (specifically framerate drops in Anime.js), accessibility (A11y), and adherence to the "Chibi Fluff" hand-drawn design tokens.
*   **Extract:** Identify reusable logic, recurring design patterns, or brand-specific constants from the conversation and move them into `MEMORY.md`.
*   **Refine:** Iterate on existing code. If a solution is "good," the agent must suggest how to make it "excellent" (e.g., optimizing a render cycle or smoothing an animation curve).
*   **Retrieve:** Before starting any task, the agent must check `CONTEXT.md` for current status and `MEMORY.md` for learned lessons/design tokens to ensure continuity.

## 3. Technology & Architecture Standards
*   **Framework:** React Native via Expo (SDK Latest).
*   **Animation Engine:** Anime.js for complex marketing sequences and vertical video generation.
*   **Styling:** Utility-first (NativeWind/Tailwind) or Styled Components, prioritizing the "hand-drawn" aesthetic.
*   **Environment:** Optimized for **Apple M5 / MLX**. Configs must be provided in **YAML** format.
*   **Modularity:** Follow **Atomic Design** principles (Atoms, Molecules, Organisms).

## 4. Agent Behavior Rules
1.  **Plan First:** Never write code without first generating a "Plan Artifact" (a step-by-step technical outline).
2.  **Zero-Hallucination:** If a library version or Expo capability is uncertain, flag it for verification.
3.  **Visual Integrity:** Always respect the "Chibi Fluff" brand. If a component looks too "generic/corporate," it must be **Refined** to fit the hand-drawn style.
4.  **State Management:** Prioritize clean, predictable data flow.

## 5. Persistence Protocol
The Agent is responsible for maintaining the "Digital Brain":
*   Update `CONTEXT.md` after every milestone.
*   Log significant technical hurdles in `JOURNAL.md`.
*   Save reusable "Golden Code" snippets in `MEMORY.md`.
